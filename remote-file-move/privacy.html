Remote File Move · Privacy Policy

Privacy Policy

Last updated: May 29, 2026

Remote File Move transfers files between two Chrome browsers using a pairing code and a WebRTC data channel.

Data collection

Remote File Move does not collect, sell, or transfer user data to the publisher or to third parties for advertising or analytics. The extension does not require an account and does not include analytics or advertising code.

File contents

File contents are sent over the WebRTC data channel between the paired browsers when the connection succeeds. File contents are not sent to the publisher. The extension does not store transferred files on a publisher server.

Signaling

The extension uses random-room signaling through https://ntfy.sh to exchange WebRTC offer/answer messages between the two browsers. Signaling messages may include WebRTC connection metadata needed to establish the peer connection, but they are not intended to contain file contents.

Local settings

The extension may store local status or preference values in Chrome local storage. These values remain on the user's browser.

Permissions

  • storage: Save local extension status/preferences.
  • notifications: Show local transfer status notifications.
  • host permission for ntfy.sh: Send and receive pairing/signaling messages for WebRTC setup.

Remote code

The extension does not load or execute remote JavaScript or WebAssembly. All extension code is included in the package submitted to the Chrome Web Store.
