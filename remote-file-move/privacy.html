Remote File Move · Privacy Policy

Privacy Policy

Last updated: May 29, 2026

Remote File Move moves allowed files between trusted Windows PCs using a Chrome extension and a separately installed native Windows helper. The extension is the Chrome control surface; the helper performs the local OS clipboard/FileDrop work needed for real file paste.

Data collection

Remote File Move does not collect, sell, or transfer user data to the publisher or to third parties. The extension does not require an account and does not include analytics, advertising, or tracking code.

Local helper and settings

The native helper may store bridge configuration, current PC side, local token, received-file state, and logs on the user's own Windows device. Chrome local storage may store local popup/status preferences. These values are used only to provide the user's own PC-to-PC bridge workflow.

File transfer data

Files are transferred only through the user's configured trusted bridge/relay. The publisher does not receive files, file names, bridge tokens, local logs, or bridge activity. Allowed transfer types are limited to png, jpg, jpeg, pdf, and zip, with a 100MB package limit in the default helper configuration.

Permissions

  • nativeMessaging: Communicate with the Remote File Move native Windows helper. This is required because Chrome extensions cannot directly set the Windows FileDrop clipboard.
  • storage: Save local popup/status preferences.
  • downloads: Save a pending bridge ZIP when the user chooses the Chrome-download fallback.
  • notifications: Show local completion/status notifications.
  • alarms: Periodically refresh helper/bridge status while Chrome is running.
  • host permissions: Connect only to trusted bridge/localhost endpoints included in the extension package for health checks and user-requested downloads.

Remote code

The extension does not load or execute remote JavaScript or WebAssembly. All extension code is included in the package submitted to the Chrome Web Store.
