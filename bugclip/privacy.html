BugClip Privacy Policy

Last updated: May 17, 2026

BugClip is a Chrome extension that helps users create local developer-ready bug packets from the active browser tab.

Information BugClip handles

BugClip may process the following information only after the user clicks the extension and chooses to create a bug packet:

  • A screenshot of the active tab or selected capture area.
  • The active page URL and page title.
  • Timestamp, user agent, viewport size, device pixel ratio, browser language, and timezone.
  • User-created annotations such as pins, boxes, arrows, text labels, and blur regions.
  • The short bug description typed by the user.
  • Click path, console/error signals, and failed-resource signals captured during reproduce mode after the user starts that mode.
  • Selected element text when the user pins a visible element, except sensitive fields BugClip identifies as private.

Local-first storage and export

BugClip does not require an account for local capture or local export.

BugClip stores local options and recent report summaries in Chrome extension storage on the user's device/browser profile. Recent local report summaries are limited to the latest 20 items.

When the user chooses an export action, BugClip can copy Markdown to the clipboard or download local PNG, JSON, Markdown, or ZIP files. ZIP exports include screenshot.png, metadata.json, and report.md.

No hosted sharing in the MVP

The current MVP does not upload bug packets to a BugClip or linqen server. Hosted share links, team inbox, history sync, and account/payment features are shown as future paid features and are not active in this local MVP.

Sensitive information handling

BugClip attempts to suggest blur regions for common sensitive fields such as password, email, phone, payment, token, one-time-code, and similarly named fields. Users can also add manual blur boxes before export.

Because screenshots can contain visible page content, users should review the preview and add blur boxes before sharing an export.

Permissions

  • activeTab: temporarily access the active tab after the user clicks the extension.
  • tabs: read the active tab ID, URL, and title for the local bug packet.
  • scripting: inject the annotation overlay into the active tab after user action.
  • storage: store local settings and recent local report summaries.
  • downloads: save local bug packet exports.
  • clipboardWrite: copy Markdown only when the user clicks the copy action.

BugClip v0.1.0 does not request broad host permissions.

Sharing and selling data

BugClip does not sell user data.

BugClip does not use user data for advertising, profiling, creditworthiness, lending, or unrelated purposes.

BugClip does not transfer user data to third parties except as required by Chrome browser APIs to perform actions requested by the user, or as required by law.

Remote code

BugClip does not use remote JavaScript or remote WebAssembly. Its extension code is included in the submitted Chrome Web Store package.

User control and deletion

Users can clear BugClip's local extension data through Chrome extension settings, browser storage controls, or by uninstalling the extension.
