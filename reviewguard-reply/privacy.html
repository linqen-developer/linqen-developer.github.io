ReviewGuard Reply

Privacy Policy

Effective date: May 14, 2026

ReviewGuard Reply helps Google Business owners draft safer review replies. The extension is designed to keep review drafting local to the user’s browser whenever possible.

Data processed by the extension

ReviewGuard Reply may read visible Google review text on supported Google Business pages so it can create draft reply suggestions, show risk warnings, and insert a selected draft after the user clicks the insert button.

Data stored locally

The extension may store the following settings in Chrome storage on the user’s device:

  • Business name and industry
  • Preferred reply tone and language setting
  • Blocked phrases and private contact/escalation line
  • Local monthly draft usage count
  • Optional local reply history, only if the user enables history retention

Users can edit settings and clear locally stored reply history from the extension settings page.

No external AI API or backend in the beta

The current beta does not send review text to an external AI API, billing backend, or analytics backend. It uses Chrome built-in AI when available and a local safe-template draft mode when built-in AI is unavailable.

No automatic posting

ReviewGuard Reply does not publish replies automatically. It only inserts a selected draft into a detected reply box after the user explicitly clicks the insert button. The user must manually review and publish the reply in Google.

Data sharing and sale

We do not sell user data. We do not transfer user data to third parties except as required to operate the extension’s narrow review-reply drafting purpose or as required by law.

Future changes

Paid plans are marked “Coming soon” in the beta. If future versions add accounts, billing, cloud AI fallback, team features, or analytics, this policy will be updated before those features are released.
